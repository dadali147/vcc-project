## 背景
V3 数据库重构已完成（13 张新表 + 7 张旧表改造）。本任务基于新表结构重构上游对接层。

## 任务范围

### 1. 渠道管理（vcc_channel）
- 实现 ChannelService：渠道 CRUD + 状态切换
- 支持多渠道配置（目前只有 YEEVCC，架构上支持扩展）
- RSA 密钥对存储到 vcc_channel 表

### 2. 卡 BIN 管理（vcc_card_bin）
- 实现 CardBinService：BIN CRUD + 状态管理
- BIN 关联渠道（channel_id）
- 默认额度配置（单日/单月/单笔）

### 3. 商户 BIN 分配（vcc_merchant_bin）
- 实现 MerchantBinService：分配/撤销 BIN 给商户
- 唯一约束：一个 BIN 给一个商户只分配一次
- 分配状态：ACTIVE/INACTIVE

### 4. YeeVCC API 客户端重构
- 基于新 vcc_channel 表动态加载 API 配置
- 支持从渠道配置读取 RSA 密钥（替代硬编码）
- 保持现有 YOP 签名逻辑不变
- 支持测试/生产环境切换

### 5. 上游接口封装
- 持卡人创建/查询
- 开卡申请
- 卡片操作（冻结/解冻/注销）
- 余额查询/充值
- 交易查询

## 技术约束
- Spring Boot 3 + MyBatis
- 包名: com.vcc.upstream, com.vcc.channel
- 状态字段用 VARCHAR code（ENABLED/DISABLED）
- 金额 DECIMAL(18,2)
- 高风险操作写审计日志
- 异常统一处理，不吞错误

## 交付物
1. Mapper XML + Java Interface（channel, card_bin, merchant_bin）
2. Service 层实现
3. Controller 层（仅管理端接口）
4. YeeVCC API 客户端适配器
5. 单元测试（Service 层核心逻辑）
