## 背景
V3 数据库重构已完成。本任务基于新表结构实现核心业务逻辑。

## 任务范围

### 1. 持卡人管理（vcc_card_holder）
- CardHolderService: CRUD + 状态管理
- 支持逻辑删除（deleted_flag）
- 一个用户可创建多个持卡人（无唯一索引）
- 持卡人字段：姓名/手机/邮箱/证件/地址

### 2. 开卡流程（vcc_card_issue_request + vcc_card_issue_item）
- CardIssueService: 完整开卡流程
  - 创建申请单（batch_no 自动生成）
  - 批量开卡（遍历 items → 调上游 → 回填结果）
  - 状态流转: PENDING → PROCESSING → COMPLETED/PARTIAL/FAILED
  - 成功/失败计数自动更新
- 开卡后自动创建 vcc_card 记录
- 卡类型区分: PREPAID（储值卡）/ BUDGET（预算卡）

### 3. 卡片管理（vcc_card）
- CardService: 查询/冻结/解冻/注销
- 卡号前6后4存储（first6/last4）
- 三要素不落库（AES 加密传输，不入 DB）
- 状态: CREATED/ACTIVE/FROZEN/CLOSED/FAILED
- 预算卡：budget_amount 额度管理

### 4. 充值业务（vcc_recharge）
- RechargeService: 充值流程
  - 充值场景区分: TOP_UP（账户充值）/ CARD_LOAD（充值到卡）
  - PREPAID 卡: 充值到卡余额
  - BUDGET 卡: 充值到账户余额（消费时扣）
  - USDT 充值: chain_type 记录链类型（TRC20/ERC20）
  - 状态: PENDING → SUCCESS/FAILED
- 手续费计算：查询 vcc_fee_config

### 5. 限额管理（vcc_limit_history）
- LimitService: 预算卡额度调整
- 每次调整记录到 vcc_limit_history
- 字段: before_value/after_value/operator/reason

### 6. 欠费管理（vcc_card_debt）
- DebtService: PREPAID 卡余额负数自动产生欠费记录
- 欠费状态: OUTSTANDING/SETTLED

## 技术约束
- Spring Boot 3 + MyBatis
- 包名: com.vcc.card, com.vcc.holder, com.vcc.recharge, com.vcc.finance
- 状态字段用 VARCHAR code
- 金额 DECIMAL(18,2)
- 高风险操作（冻结/注销/调账）必须写审计日志
- 事务完整性：开卡失败不影响其他卡片
- KYC 门控：未通过 KYC 不可开卡（抛业务异常）
