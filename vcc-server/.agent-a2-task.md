## SQL 编写规范
1. 状态字段用 VARCHAR/CHAR 存英文 code
2. 所有核心表有 created_at DATETIME DEFAULT CURRENT_TIMESTAMP, updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
3. JSON 字段用 JSON 类型（MySQL 8.0）
4. PREPAID 卡余额不加 NOT NULL 约束限制正数
5. 逻辑删除用 del_flag CHAR(1) DEFAULT '0'
6. 所有金额 DECIMAL(18,2)
7. snake_case 命名
8. ENGINE=InnoDB, DEFAULT CHARSET=utf8mb4, COLLATE=utf8mb4_unicode_ci
9. 加 COMMENT
10. ALTER TABLE 用 IF NOT EXISTS 或注释说明

## 关键业务硬约束
- 用户状态: ENABLED / DISABLED 两态
- KYC 状态: NOT_SUBMITTED / PENDING / APPROVED / REJECTED / LOCKED
- PREPAID 卡允许负数余额
- 一个 BIN 给一个商户只能分配一次(UNIQUE约束)
- 核心对象不做物理删除
- 主账户充值第一版仅 USDT
