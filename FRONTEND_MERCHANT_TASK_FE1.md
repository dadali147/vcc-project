## 一、任务目标
从零初始化 VCC 虚拟卡项目【商户端】的前端框架，并完成登录/注册及最核心的【卡片资产】相关页面体系。
核心目标包括：
1. 搭建基础框架体系 (Vue 3 + Vite + Element Plus + TailwindCSS)。
2. 实现 `vcc-ui-design` 设计规范，集成 Kimoox Logo 色系。
3. 完成路由与权限拦截骨架。
4. 交付 4 个核心页面：登录/注册页、Dashboard (首页)、卡片控制台 (列表/详情)、用卡人管理。

## 二、必读输入约束
1. **全局 UI 规范**：你必须严格遵循 `~/.openclaw/workspace/skills/vcc-ui-design/SKILL.md` (已由 Gemini 3.1 Pro 重新设计)。绝对禁止使用毛坯默认样式。必须融入深蓝色/紫色系 (Indigo/Slate) 基调，强调质感、大圆角与数据右对齐。
2. **原型定位**：禁止照搬原型图。原型仅作为“需要哪些字段和按钮”的功能大纲。你的布局、交互必须符合现代 SaaS (如 Stripe/Vercel) 的高级感。
3. **接口约定**：参考《VCC 商户端接口能力清单 v1》。当前后端 API 正在联调，所有数据获取请**先写好请求与模型定义**，但底层允许配置 Mock 数据拦截以便立刻展示。

## 三、你需要负责的实现范围
你只需在 `vcc-web/merchant` 目录下完成以下事项：

### 1️⃣ 基建初始化
- 初始化项目: `npm create vue@3` 或类似工具。
- 配置 TailwindCSS (用于辅助原子类) 与 Element Plus (用作核心组件库但必须覆盖其默认土气主题)。
- 配置 Vue Router (支持懒加载)。
- 配置 Pinia (全局状态包含 `userStore`, `cardStore`)。
- 封装 Axios 请求库 (统一包含 Token 注入、401/403 拦截重定向、全局错误抛出)。

### 2️⃣ 核心页面
**Page 1: 登录与注册与找回密码**
- 居中卡片布局，使用 Kimoox 品牌理念。
- 支持邮箱+密码登录。
- 表单必须有严格校验 (Blur 校验为主)。

**Page 2: 首页 Dashboard**
- 数据卡片：包含总可用余额 (USD)、本月累计开卡、激活卡数量等。
- 近期交易骨架屏：简易列表。

**Page 3: 卡片控制台 (核心)**
- **卡片列表**：带过滤区 (持卡人/卡状态/币种)。卡号采取类似 `428888 **** 1234` 遮盖显示。
- **卡片详情操作联动**：通过抽屉 (Drawer) 或卡内联动展示卡的独立交易明细 (Transactions) 和操作记录。
- **动作按钮**：冻结/解冻、注销 (需危险二次确认样式)。

**Page 4: 用卡人管理 (持卡人)**
- 显示商户下的归属用卡人列表 (包含可用额度限制)。
- 创建新的用卡人弹窗。
