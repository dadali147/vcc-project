# 账户资金与费用基线 v1

## 1. 文档定位

本文件用于统一账户余额、卡余额、预算额度、冻结金额、手续费、开卡费、月费、退款回退等账务规则，作为业务唯一真相源之一。

## 2. 核心卡类型

### 2.1 PREPAID（储值卡）

- 资金路径：账户 → 卡内 → 消费
- 交易手续费：从卡内扣除

### 2.2 BUDGET（预算卡）

- 资金路径：账户 → 设置额度 → 消费时账户扣费
- 交易手续费：从账户扣除

## 3. 核心资金对象

| 对象 | 说明 |
|------|------|
| 账户余额 | 商户账户可用资金 |
| 卡余额 | 储值卡卡内余额 |
| 预算额度 | 预算卡当前允许消费上限 |
| 冻结金额 | 已冻结待结算或待释放金额 |
| 手续费 | 开卡费、月费、交易手续费等 |

## 4. 费用扣取规则

| 费用类型 | 储值卡 | 预算卡 |
|---------|--------|--------|
| 开卡费 | 从用户账户扣 | 从用户账户扣 |
| 月费 | 从用户账户扣 | 从用户账户扣 |
| 交易手续费 | 从卡内扣 | 从用户账户扣 |

## 5. 关键业务场景口径

### 5.1 开卡

- 前提：账户余额满足开卡费用要求
- 扣费位置：用户账户
- 若失败：未成功开卡则不应形成成功扣费结论，需按实际结果回退或标记失败

### 5.2 充值

- 充值目标：储值卡卡余额或账户资金体系中的可用资金
- 充值成功后，应更新对应余额并保留记录
- 充值失败不得口头算成功，必须明确失败态和回退口径

### 5.3 消费

- 储值卡：优先从卡余额扣款，手续费从卡内扣
- 预算卡：消费时从账户扣款，额度用于控制消费上限
- 消费处理中、成功、失败应区分展示

### 5.4 退款

- 应按原资金路径或正式业务规则回退
- 退款回退后必须更新对应余额/额度/记录
- 若存在手续费不退、部分退回等情况，必须在后续版本补明确口径

### 5.5 销卡 / 注销

- 应明确卡注销时剩余资金去向
- 若有可退余额、冻结金额、未结清交易，需按正式规则处理
- 不允许在资金去向未定义时直接口头宣称"已销卡完成"

## 6. 状态与记录要求

- 所有资金相关动作必须有记录
- 必须保留：操作类型、操作时间、对象、金额、结果状态
- 前端展示口径应与业务规则一致，不得自行解释账务变化

## 7. 当前版本边界

本版本先收首轮统一规则，不展开到全部手续费细分场景；后续版本需继续细化退款、撤销、拒付、冻结释放等特殊账务处理。

## 8. 验收标准

- 明确 PREPAID / BUDGET 两类卡的资金路径
- 明确开卡费、月费、交易手续费从哪里扣
- 覆盖开卡、充值、消费、退款、销卡五类关键场景
- 可作为后续接口、数据库、页面展示与测试用例的统一依据
