# 01.4-页面基线-页面清单与字段

## 来源文档

- VCC 商户端页面清单总表 v1.0
- VCC 页面字段草案 v1.0
- VCC 页面原型说明 v1.0
- 02.1-内部系统接口契约 v1
- 02.2-接口契约交叉校验记录 v1

## 定位

本文件用于承载页面级业务真相源，统一页面清单、页面字段、页面职责、页面边界与展示形态。

## 使用规则

- 页面字段必须直接对齐正式接口字段，不允许页面自己造命名。
- 聚合展示字段可存在，但必须明确"展示名"和"真实字段"的对应关系。
- 页面状态、按钮显隐、流程跳转必须绑定正式状态字典，不允许写死在前端组件里。
- 页面如需新增字段，必须先补本文件，再改接口或原型。

## 页面字段对齐结果（第一版）

### 1. 注册页面

**页面职责**

- 完成商户主账户注册
- 获取注册结果并引导下一步（KYC / 待审核 / 驳回）

**页面输入字段**

| 页面字段 | 对齐接口字段 | 必填 | 说明 |
|----------|-------------|------|------|
| 商户名称 | merchantName | 是 | 注册主体名称 |
| 联系人姓名 | contactName | 是 | 主联系人 |
| 邮箱 | email | 是 | 登录账号与唯一标识 |
| 密码 | password | 是 | 注册密码 |
| 确认密码 | confirmPassword | 是 | 必须与 password 一致 |
| 国家 | country | 否 | 注册补充信息 |
| 邀请码 | inviteCode | 否 | 渠道归因 |
| 邮箱验证码 | emailCaptcha | 是 | 统一字段名，不再分叉成 code / captchaCode |
| 同意协议 | agreeProtocol | 是 | 布尔字段，必须为 true |

**页面结果字段**

| 页面字段 | 对齐接口字段 | 说明 |
|----------|-------------|------|
| 用户 ID | userId | 注册成功后返回 |
| 商户 ID | merchantId | 注册成功后返回 |
| 注册状态 | registerStatus | 决定后续引导 |
| 下一步 | nextStep | 决定跳 KYC / 待审核等 |
| 是否需要 KYC | needKyc | 首页或结果页引导 |
| 登录凭证 | token | 支持自动登录场景时返回 |

**已收口规则**

- 注册验证码字段统一使用 emailCaptcha。
- 页面协议勾选必须真实落到 agreeProtocol，不能只做前端态。

**当前缺口**

- "发送邮箱验证码"接口尚未在 02.1 单独成文。
- nextStep 枚举值仍未正式收口。

### 2. 登录页面

**页面职责**

- 完成商户端或管理端登录
- 初始化用户权限、菜单、角色信息

**页面输入字段**

| 页面字段 | 对齐接口字段 | 必填 | 说明 |
|----------|-------------|------|------|
| 账号 | email | 是 | 前端文案可显示邮箱/账号，但接口字段统一为 email |
| 密码 | password | 是 | 登录密码 |
| 图形验证码 | captcha | 条件必填 | 启用风控或验证码时必填 |
| 登录类型 | loginType | 是 | merchant / admin |

**页面结果字段**

| 页面字段 | 对齐接口字段 | 说明 |
|----------|-------------|------|
| 登录凭证 | token | 会话凭证 |
| 用户 ID | userId | 当前登录用户 |
| 用户类型 | userType | 商户主账户 / 子账户 / 管理端 |
| 角色集合 | roles | 角色控制 |
| 权限集合 | permissions | 按钮级权限控制 |
| 昵称 | nickName | 顶部用户信息 |

**已收口规则**

- 登录账号统一走 邮箱，不要再拆邮箱 / 手机 / 账号三个字段口径。
- 登录验证码字段统一走 captcha，与注册页 emailCaptcha 区分。

**当前缺口**

- 登录响应是否要同时回传 kycStatus / registerStatus，仍未写死。

**页面职责**

- 展示 KYC 审核结果与拒绝原因

| 页面字段 | 对齐接口字段 | 必填 | 说明 |
|----------|-------------|------|------|
| 姓名 | realName | 是 | 个人实名 |
| 证件类型 | idType | 是 | 与状态字典后续对齐 |
| 证件号 | idNumber | 是 | 敏感字段 |
| 国籍 | nationality | 是 | 基础身份信息 |
| 证件正面 | idFrontImage | 是 | 图片上传 |
| 证件反面 | idBackImage | 是 | 图片上传 |
| 自拍照 | selfieImage | 是 | 活体 / 校验资料 |
| 问卷 | questionnaire | 是 | 当前仍为聚合字段 |

**企业 KYC 提交字段**

| 页面字段 | 对齐接口字段 | 必填 | 说明 |
|----------|-------------|------|------|
| 企业名称 | companyName | 是 | 企业主体 |
| 企业编号 | companyNo | 是 | 主体证照号 |
| 国家 | country | 是 | 注册国家 |
| 注册文件 | incorporationDoc | 是 | 企业文件 |
| 法人姓名 | legalPersonName | 是 | 法人信息 |
| 法人证件信息 | legalPersonIdInfo | 是 | 当前为聚合对象 |
| 问卷 | questionnaire | 是 | 当前为聚合字段 |

**结果字段**

| 页面字段 | 对齐接口字段 | 说明 |
|----------|-------------|------|
| KYC 状态 | kycStatus | 提交后与结果页都使用 |
| 拒绝原因 | rejectReason | 驳回时展示 |
| 最后提交时间 | lastSubmitTime | 结果页展示 |
| 审核时间 | reviewTime | 结果页展示 |
| 审核备注 | reviewRemark | 结果页展示 |

**已收口规则**

- 提交字段与结果字段必须拆开建模，rejectReason 不能混进表单模型。
- 个人 / 企业 KYC 都保留 questionnaire 聚合字段，但后续必须继续拆细。

**当前缺口**

- 图片字段格式（URL / fileToken / attachmentId）未定。
- questionnaire、legalPersonIdInfo 还未拆成正式字段表。

- 在详情页查看状态、卡片数量、最近操作记录

**列表字段**

|----------|-------------|------|

| 手机号 | mobile | 联系方式 |
| 邮箱 | email | 联系方式 |
| 状态 | status | 后续建议统一为 holderStatus |
| 关联卡片数 | relatedCardCount | 统计字段 |
| 创建时间 | createTime | 列表展示 |

**新增 / 编辑字段**

| 页面字段 | 对齐接口字段 | 必填 | 说明 |
|----------|-------------|------|------|
| 名 | firstName | 是 | 真实字段 |
| 姓 | lastName | 是 | 真实字段 |
| 手机号 | mobile | 是 | 联系方式 |
| 邮箱 | email | 是 | 联系方式 |
| 国家 | country | 是 | 基础地址信息 |
| 地址 | address | 是 | 地址信息 |
| 城市 | city | 是 | 地址信息 |
| 州 / 省 | state | 是 | 地址信息 |
| 邮编 | postCode | 是 | 地址信息 |
| 证件类型 | idType | 是 | 证件信息 |
| 证件号 | idNumber | 是 | 敏感字段 |

**详情字段要求**

- 基础信息
- 状态信息
- 名下卡片统计
- 最近操作记录（直接对接操作记录接口）

**已收口规则**

- 最近操作记录不能让前端自己拼，必须对接正式操作记录接口。

**当前缺口**

- 持卡人详情返回字段还过于抽象，未展开为正式字段清单。
- status 建议升级为 holderStatus，避免与其他对象通用 status 混淆。

### 6. 开卡页面

**页面职责**

- 提交开卡申请
- 展示提交结果并引导进入任务查询 / 开卡记录

**页面输入字段**

| 页面字段 | 对齐接口字段 | 必填 | 说明 |
|----------|-------------|------|------|
| 持卡人 | holderId | 是 | 关联持卡人 |
| BIN | cardBinId | 是 | 选择发卡产品 |
| 卡类型 | cardType | 是 | PREPAID / BUDGET |
| 卡名称 | cardName | 是 | 页面展示名 |
| 预算金额 | budgetAmount | 条件必填 | 仅 BUDGET 卡必填 |
| 币种 | currency | 是 | 结算币种 |
| 消费范围 | spendScope | 否 | 当前枚举未定 |

**页面结果字段**

| 页面字段 | 对齐接口字段 | 说明 |
|----------|-------------|------|
| 任务 ID | taskId | 提交后优先返回 |
| 申请状态 | applyStatus | 开卡业务状态 |
| 预计完成时间 | estimatedFinishTime | 结果提示 |

**已收口规则**

- budgetAmount 仅在 cardType = BUDGET 时必填，页面必须联动显隐。
- 开卡结果不要假定同步返回 cardId，优先走 taskId 查询链路。

**当前缺口**

- spendScope 枚举未定义。
- BIN 下拉项来源接口尚未成文。

**页面输入字段**

| 页面字段 | 对齐接口字段 | 必填 | 说明 |
|----------|-------------|------|------|

| 币种 | currency | 是 | 充值币种 |
| 商户订单号 | orderNo | 是 | 商户内唯一 |
| 备注 | remark | 否 | 业务备注 |
| 扣款账户 | sourceAccountId | 是 | 资金来源 |

**充值结果字段**

| 页面字段 | 对齐接口字段 | 说明 |
|----------|-------------|------|
| 商户订单号 | orderNo | 页面结果回显 |
| 上游订单号 | upstreamOrderNo | 上游追踪 |
| 充值状态 | status | 后续建议升级为 rechargeStatus |
| 充值后余额 | balance | 结果展示 |

**辅助展示字段**

**已收口规则**

- 页面必须明确"账户余额"和"卡片余额"两个口径，不能混着展示。

**当前缺口**

- orderNo 由前端生成还是后端生成，尚未定稿。
- 充值失败原因字段未在 02.1 正式写明。

## 仪表盘首页

**页面职责**：展示商户账户总览、KYC 进度、卡片/持卡人/余额核心指标；承接待办提醒、异常提示、快捷操作入口。

**核心展示字段**：merchantId、merchantName、registerStatus、kycStatus、availableBalance、frozenBalance、cardCount、activeCardCount、holderCount、pendingTaskCount、lastLoginTime。

**已收口规则：**

- 首页指标字段可以聚合返回，但必须保留真实字段名，不允许前端二次造词。
- registerStatus / kycStatus 直接驱动待办区，不允许写死文案。

**当前缺口：**

- 首页聚合接口尚未在 02.1 独立成文。
- pendingTaskCount 的口径需明确是否包含导出任务、开卡任务、充值任务。

## 卡片列表页面

**页面职责**：分页查看卡片，支持筛选、搜索、状态标签与快捷操作；作为进入卡片详情、充值、冻结/解冻等动作的主入口。

**筛选字段**：cardType、cardStatus、holderId、cardBinId、keyword。

**列表字段**：cardId、cardName、cardType、holderName、cardNoMask、cardStatus、currency、balance、budgetAmount、availableAmount、createTime。

**已收口规则：**

- 卡片列表状态列统一使用 cardStatus，不再裸用 status。
- balance / budgetAmount / availableAmount 允许同表展示，但前端必须根据 cardType 区分文案和含义。

**当前缺口：**

- 列表快捷操作（冻结、解冻、充值、查看详情）与权限点矩阵还未补。
- keyword 的匹配范围仍未在 02.1 写死。

## 开卡记录页面

**页面职责**：查询开卡申请历史与最终结果；提供 taskId 维度的结果追踪与失败原因查看。

**筛选字段**：taskId、holderId、applyStatus、taskStatus、timeRange。

**列表字段**：taskId、holderName、cardType、applyStatus、taskStatus、cardId、rejectReason、errorCode、createdTime、finishedTime。

**已收口规则：**

- 开卡记录页必须双状态展示：applyStatus + taskStatus，不能只展示一个"处理中"。
- SUCCESS 但业务失败时，仍需通过 applyStatus / rejectReason / errorCode 呈现真实结果。

**当前缺口：**

- 开卡记录查询接口尚未在 02.1 独立编号。
- cardId 是否在列表直接返回还是点详情再查，仍未写死。

## 记录中心

**页面职责**：承接交易记录、充值记录、操作记录等统一检索入口；让商户按对象、动作、时间快速回溯业务轨迹。

**一级分类字段**：recordType、targetType、targetId、timeRange。

**统一列表字段（按类型裁剪）**：recordId/txnId/orderNo、recordType、transactionStatus/rechargeStatus/resultStatus、amount、currency、actionType、operatorName、requestTime/txnTime/finishTime、traceId。

**已收口规则：**

- 记录中心可以做统一壳页，但底层数据源不能强行合成一个接口。
- recordType 仅是页面维度分类，不可替代底层正式业务状态字段。

**当前缺口：**

- recordType 枚举尚未正式收口。
- 统一下载/导出能力与下载中心的关系还没补说明。

## 下载中心

**页面职责**：展示导出任务、文件生成结果、下载入口与过期状态；统一承接账单导出、报表导出等文件结果。

**列表字段**：fileId、fileName、fileType、language、downloadStatus、taskStatus、downloadUrl、createTime、expireTime。

**已收口规则：**

- 下载中心页面至少要同时识别 taskStatus 与 downloadStatus；前者看生成过程，后者看文件是否可下。
- downloadUrl 仅在 downloadStatus = READY 且未过期时可点。

**当前缺口：**

- fileType 枚举、language 枚举尚未正式定义。
- 手动删除文件后的历史展示策略未定。

## 子账户管理页面

**页面职责**：查看、创建、编辑、禁用、重置子账户；配置角色权限与数据可见范围。

**列表字段**：subAccountId、loginEmail、userName、subAccountStatus、roleCode、roleName、dataScopeType、lastLoginTime、createTime。

**新增/编辑字段**：userName、loginEmail、mobile、roleIds、dataScopeType、dataScopeValue、remark、initPasswordMode。

**已收口规则：**

- 页面状态字段统一使用 subAccountStatus，不再裸用 status。
- 角色配置与数据范围配置必须同一表单提交，避免中间态越权。

**当前缺口：**

- 权限树接口、menuKeys / permissionKeys 结构还未展开到页面字段级。
- 子账户详情页摘要字段还未补完整。

## 消息通知页面

**页面职责**：查看系统通知、业务提醒、审核结果、任务结果等消息；支持按类型、已读状态、时间过滤。

**列表字段**：messageId、messageType、title、summary、readStatus、targetType、targetId、sendTime、readTime。

**已收口规则：**

- 消息通知页必须有 readStatus 和 targetType/targetId，不然无法闭环到页面跳转。
- messageType / readStatus 当前先作为页面字段骨架，后续应进入 01.3 正式状态/枚举收口。

**当前缺口：**

- 消息通知相关接口尚未在 02.1 成文。
- messageType、readStatus 枚举和值域未定稿。

## 个人设置 / 安全设置

**页面职责**：维护当前登录用户个人资料、安全信息、密码修改、登录设备与安全提醒；承接账户安全相关高频操作。

**个人资料字段**：userId、nickName、userName、email、mobile、userType、roles。

**安全设置字段**：oldPassword、newPassword、confirmPassword、lastLoginTime、lastLoginIp、needResetPassword。

**已收口规则：**

- 个人设置与安全设置可以同页分区，但密码修改字段不能和资料更新字段混成一个提交模型。
- needResetPassword 应直接驱动改密弹窗或跳转，不允许前端自行猜测。

**当前缺口：**

- 个人资料更新接口、登录设备/安全日志接口尚未在 02.1 成文。
- lastLoginIp 是否允许商户端直接展示，需结合安全策略再定。

## 聚合对象字段展开（第一版）

### holderInfo（卡片详情中的持卡人聚合对象）

**字段清单**：holderId、firstName、lastName、holderName、email、mobile、holderStatus、idType、idNumberMask、country。

**字段说明：**

- holderName 为展示字段，可由 firstName + lastName 拼装，但接口允许直接返回展示值，避免前端重复拼接。
- idNumberMask 仅允许脱敏展示，不回传完整证件号。
- holderStatus 必须直接使用 01.3 正式枚举，不再裸用 status。

**页面使用位置**：卡片详情基本信息区、开卡记录跳转、持卡人详情侧栏。

**规则补充：**

- holderInfo 仅承载展示与关联跳转所需摘要；完整可编辑资料仍应走持卡人详情接口。
- 卡片详情页不应通过 holderInfo 回传完整身份证号、详细住址等高敏字段。

### balanceInfo（卡片余额聚合对象）

**字段清单**：accountBalance、cardBalance、availableAmount、frozenAmount、budgetAmount、availableBudget、currency、lastUpdateTime。

**字段说明：**

- accountBalance：商户账户维度余额，用于说明充值资金来源或预算总承载能力。
- cardBalance：卡片当前余额，主要适用于 PREPAID 卡。
- availableAmount：当前可用金额，适用于卡片消费可用口径。
- frozenAmount：冻结金额。
- budgetAmount：预算总额，主要适用于 BUDGET 卡。
- availableBudget：剩余可用预算，主要适用于 BUDGET 卡。
- currency：币种。
- lastUpdateTime：余额快照更新时间。

**页面使用位置**：卡片详情余额区、充值结果联动区。

**规则补充：**

- PREPAID 卡优先展示 cardBalance / availableAmount；BUDGET 卡优先展示 budgetAmount / availableBudget，前端不得把两类卡文案混用。
- 如 accountBalance 非本接口强依赖，可作为可选字段返回，但一旦返回必须明确口径。

### feeInfo（卡片费用聚合对象）

**字段清单**：openCardFee、monthlyFee、transactionFeeRate、transactionFeeFixed, feeCurrency、chargeMode、lastChargeTime。

**字段说明：**

- openCardFee：开卡费。
- monthlyFee：月费。
- transactionFeeRate：交易手续费率。
- transactionFeeFixed：交易固定手续费。
- feeCurrency：费用币种。
- chargeMode：扣费方式，建议枚举 ACCOUNT / CARD。
- lastChargeTime：最近一次费用扣取时间。

**页面使用位置**：卡片详情费用区、开卡说明区。

**规则补充：**

- chargeMode 必须显式返回，用来承接"储值卡交易手续费卡内扣、预算卡交易手续费账户扣"的业务规则。
- feeInfo 用于展示当前生效费用摘要；历史费率变更记录应走独立费率/账单接口。

### limitInfo（卡片限额聚合对象）

**字段清单**：dailyLimit、monthlyLimit、singleLimit、usedTodayAmount、usedMonthAmount、remainingTodayAmount、remainingMonthAmount、limitCurrency、budgetStatus、effectiveTime。

**字段说明：**

- dailyLimit：单日限额。
- monthlyLimit：单月限额。
- singleLimit：单笔限额。
- usedTodayAmount / usedMonthAmount：已使用额度。
- remainingTodayAmount / remainingMonthAmount：剩余额度。
- limitCurrency：限额币种。
- budgetStatus：预算状态，适用于预算卡或共享卡预算场景。
- effectiveTime：当前限额生效时间。

**页面使用位置**：卡片详情限额区、共享卡限额页摘要。

**规则补充：**

- limitInfo 是当前生效快照，不等于历史调额记录；历史记录需走限额历史查询接口。
- BUDGET 卡、共享卡优先展示 budgetStatus 与额度快照；PREPAID 卡如无预算能力可返回 null 或不返回预算相关字段。

### questionnaire（KYC 问卷聚合对象）

**字段清单**：occupation、sourceOfFunds、monthlyTurnoverLevel、expectedUsageScenario、businessNature、websiteUrl、socialMediaUrl、contactPurpose。

**字段说明：**

- occupation：个人职业或企业联系人职业。
- sourceOfFunds：资金来源。
- monthlyTurnoverLevel：预估月流水级别。
- expectedUsageScenario：预期用卡/收付款场景。
- businessNature：企业业务性质。
- websiteUrl：企业官网。
- socialMediaUrl：社媒主页，可选。
- contactPurpose：开户/发卡目的说明。

**页面使用位置**：个人 KYC、企业 KYC 表单与结果回显。

**规则补充：**

- questionnaire 现在至少拆到字段级，不再允许整包黑盒 object 不做说明。
- 不同主体（个人/企业）可允许部分字段为空，但字段名保持统一；如需再细分，后续补主体差异表，不要临时改名。

### legalPersonIdInfo（企业 KYC 中法人证件聚合对象）

**字段清单**：legalPersonName、idType、idNumberMask、nationality、idFrontImage、idBackImage、selfieImage、expireDate。

**字段说明：**

- legalPersonName：法人姓名。
- idType：证件类型。
- idNumberMask：法人证件号脱敏值。
- nationality：国籍。
- idFrontImage / idBackImage：证件正反面材料。
- selfieImage：法人活体或手持材料。
- expireDate：证件有效期。

**页面使用位置**：企业 KYC 表单、KYC 结果详情。

**规则补充：**

- legalPersonIdInfo 为展示与提交聚合对象；接口内部可以继续拆字段，但页面和契约层必须至少看到这一级明细。
- idNumberMask 仅用于结果展示；提交场景如必须收完整证件号，应通过独立真实字段 idNumber 入参承接，并按安全规则处理。
