# 页面与业务流程映射 v1

## 1. 文档定位

本文件用于把核心页面、页面动作、业务流程节点、状态展示之间建立正式映射，作为前端设计、交互说明、测试用例和业务流程文档之间的桥梁。

## 2. 映射原则

- 每个核心页面至少对应一个业务流程节点
- 每个关键按钮必须能映射到明确业务动作
- 页面状态展示必须能映射到统一状态字典
- 页面操作结果应能落到记录、状态或结果物

## 3. 核心映射（首版）

### 3.1 登录/注册页

- **页面动作**：登录、注册、找回
- **对应流程**：账户接入与身份确认流程
- **结果**：进入账户体系或返回错误提示

### 3.2 KYC 认证页

- **页面动作**：提交个人/企业认证资料
- **对应流程**：KYC 提交 → 审核中 → 通过/拒绝
- **状态映射**：待提交 / 审核中 / 通过 / 拒绝

### 3.3 用卡人管理页

- **页面动作**：新增、编辑、查看持卡人
- **对应流程**：持卡人信息创建与维护
- **结果**：生成或更新持卡人对象

### 3.4 卡片列表页

- **页面动作**：查询、筛选、进入详情、发起开卡
- **对应流程**：卡片查询 / 开卡入口
- **状态映射**：卡片状态、余额/额度展示

### 3.5 卡片详情页

- **页面动作**：查看基本信息、交易明细、操作记录、充值/冻结/销卡等
- **对应流程**：卡片生命周期管理
- **状态映射**：卡状态、交易状态、操作记录

### 3.6 开卡申请页

- **页面动作**：填写信息、确认提交
- **对应流程**：开卡申请 → 费用校验 → 上游受理 → 结果返回
- **状态映射**：待提交 / 处理中 / 成功 / 失败

### 3.7 交易明细页

- **页面动作**：查询、筛选、导出
- **对应流程**：交易记录查询与导出
- **状态映射**：交易状态、退款状态

### 3.8 下载中心

- **页面动作**：发起导出、查看生成结果、下载文件
- **对应流程**：导出任务创建 → 处理中 → 成功/失败
- **状态映射**：导出任务状态

## 4. 当前版本边界

本版先覆盖核心页面，不展开到每个按钮级字段；后续版本应继续细化到页面级动作矩阵。

## 5. 验收标准

- 核心页面至少映射到一个业务流程节点
- 页面动作能够映射到明确业务动作
- 页面状态能够映射到状态字典
- 可作为前端页面设计与业务流程说明的桥梁文档
