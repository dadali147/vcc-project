# 核心对象与数据口径基线 v1

## 1. 文档定位

本文件用于统一 VCC 业务中的核心对象定义、关键字段、关系边界和命名口径，作为接口契约、数据库设计、页面展示的共同上游依据。

## 2. 核心对象清单

### 2.1 账户（Account）

- **定义**：商户维度的主资金与业务归属主体
- **关键标识**：`account_id`
- **关系**：一个账户可关联多个子账户、持卡人、卡片、交易、充值记录

### 2.2 子账户（Sub Account）

- **定义**：主账户下的协作使用者或受控业务角色
- **关键标识**：`sub_account_id`
- **关系**：隶属于一个主账户，不独立脱离主账户存在

### 2.3 持卡人（Card Holder）

- **定义**：用于承接开卡主体信息的人或实体
- **关键标识**：`holder_id`
- **关系**：一个账户可有多个持卡人；一个持卡人可关联多张卡

### 2.4 卡片（Card）

- **定义**：实际承载消费能力的虚拟卡对象
- **关键标识**：`card_id`
- **核心属性**：卡类型、状态、余额/额度、所属账户、所属持卡人

### 2.5 预算组 / 预算卡（Budget Group / Budget Card）

- **定义**：用于预算控制的资金约束对象与卡片类型表达
- **口径**：预算卡受预算额度控制，预算组关系在后续版本需进一步明确是否作为独立对象存在

### 2.6 交易（Transaction）

- **定义**：消费、退款、撤销等资金变动或支付事件记录
- **关键标识**：`transaction_id`
- **关系**：关联卡片、账户、金额、币种、状态、时间

### 2.7 充值（Recharge）

- **定义**：对账户或卡进行资金补充的业务记录
- **关键标识**：`recharge_id`
- **关系**：关联目标对象、金额、状态、操作人、时间

### 2.8 调额（Limit Adjustment）

- **定义**：对预算额度或限制项进行变更的记录
- **关键标识**：`adjustment_id`
- **关系**：关联卡片/预算对象、变更前后数值、操作人、时间、结果

### 2.9 导出任务（Export Task）

- **定义**：报表、交易、充值、对账单等导出动作的任务记录
- **关键标识**：`export_task_id`
- **关系**：关联导出类型、筛选条件、发起人、状态、结果文件

## 3. 核心关系口径

- 一个账户可有多个子账户
- 一个账户可有多个持卡人
- 一个账户可有多张卡
- 一个持卡人可关联多张卡
- 一张卡归属于一个账户，并通常关联一个持卡人
- 一个账户与多笔交易、充值、导出任务相关联

## 4. 字段口径原则

- 每个核心对象必须有唯一标识
- 状态字段必须引用统一状态字典
- 页面展示字段命名应与对象定义一致
- 接口字段与数据库字段允许命名风格差异，但业务含义必须一致

## 5. 当前版本边界

本版先收口业务对象定义与关系，不展开到完整字段级字典；后续应与接口契约、数据库设计、页面字段文档逐项交叉校验。

## 6. 验收标准

- 覆盖账户、子账户、持卡人、卡片、交易、充值、调额、导出任务等核心对象
- 明确关键标识与对象关系
- 对预算组/预算卡给出当前正式口径
- 可直接作为 02 接口契约与数据库设计的映射依据
