# 术语与命名统一基线 v1

## 1. 文档定位

本文件用于统一 VCC 项目中的高频业务术语、正式名称、别名、废弃叫法和对内对外展示口径，避免文档、页面、接口、数据库命名互相打架。

## 2. 统一原则

- 一个核心概念只保留一个正式名称
- 历史叫法可以保留，但必须标记为别名或废弃叫法
- 对外展示名称与对内实现名称可不同，但必须建立映射

## 3. 核心术语（首版）

| 术语 | 说明 |
|------|------|
| 储值卡 | 正式名称，对应 PREPAID |
| 预算卡 | 正式名称，对应 BUDGET |
| 共享卡 | 当前应与预算卡关系明确，未确认前不得与预算卡随意混用 |
| 主账户 | 商户主控制账户 |
| 子账户 | 主账户下的受控协作账户 |
| 账户余额 | 账户维度可用资金 |
| 卡余额 | 储值卡卡内资金 |
| 预算额度 | 预算卡消费上限 |
| 持卡人 | 承接开卡主体信息的业务对象 |
| 卡片 | 实际承载消费能力的虚拟卡对象 |
| 交易 | 消费、退款、撤销等支付事件记录 |
| 充值 | 资金补充动作 |
| 调额 | 预算或限制项变更 |
| 导出任务 | 报表、交易、充值、对账等导出动作记录 |
| KYC | 认证审核流程对象 |

## 4. 使用要求

- 页面文案优先使用正式名称
- 接口/数据库若因历史原因保留旧字段名，需在文档中注明映射
- 新文档禁止新增未定义术语直接进入正式基线

## 5. 当前版本边界

本版先收首轮高频术语，后续版本需继续扩展到至少 20 个高频术语并与页面、接口、数据库逐项对齐。

## 6. 验收标准

- 列出正式名称、别名/废弃叫法边界
- 说明对内/对外展示口径
- 与状态字典、页面字段、对象口径文档不冲突
- 可作为后续命名统一检查依据
